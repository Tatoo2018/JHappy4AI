🤖 JHappy AI Exporter 用户指南

JHappy AI Exporter 是一款专用工具，旨在以“AI (LLM) 最易理解的格式”导出 Eclipse 项目资产 [cite: 1, 35, 39]。它可以将多个源代码文件合并为一个结构化的文本文件，从而显著提高为 ChatGPT、Gemini、Claude 等提供上下文的效率 [cite: 35, 39]。

基本用法

只需点击几次，即可将分析所需的全部代码集成为一个文件。

  1. 选择: 在包资源管理器（Package Explorer）等视图中，右键点击目标项目或目录 [cite: 26, 35]。
  2. 执行: 从菜单中选择 🤖 合并源代码并输出给 AI [cite: 1, 35]。
  3. 调整: 将显示运行设置对话框 [cite: 1]。您可以针对本次操作调整过滤器或容量限制。
  4. 保存: 点击 OK 并指定保存位置，即可生成合并后的 source_context.txt [cite: 1, 35]。

文件与目录过滤

通过排除 AI 无需处理的数据（如二进制文件或构建产物），可以节省 Token 并提高分析精度 [cite: 37]。

允许的扩展名 (白名单)

指定如 java, py, js 时，仅收集匹配的文件 [cite: 37]。如果留空，则包含所有文件。

排除的扩展名 (黑名单)

指定如 class, jar, exe 等分析中不需要的扩展名进行排除 [cite: 37]。

目录排除: 您可以按名称指定 target, bin, .git 等目录，使其在递归搜索中被跳过 [cite: 37]。

输出容量 (MB) 限制

考虑到 AI 服务的输入限制（上下文窗口），您可以限制输出文件的最大大小 [cite: 1]。

  • 以 MB 为单位指定: 在运行对话框中输入 2，则会在达到约 2MB 时自动停止写入 [cite: 1]。
  • 安全截断: 达到容量限制时，会在文件末尾插入警告消息，明确标识处理截止的文件 [cite: 1]。
  • 无限制输出: 输入 0 即可无限制地输出项目中的所有内容 [cite: 1]。

字符编码与输出格式

自动统一为 UTF-8:
即使 Eclipse 项目中的各文件使用 Shift-JIS 或 EUC-JP 编码，插件也会根据 Eclipse 的设置正确读取，并全部转换为 UTF-8 后合并为一个文件 [cite: 1]。这有效防止了 AI 端的乱码问题。
