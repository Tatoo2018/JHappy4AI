Basic Usage

Collect all necessary code for analysis into a single file with just a few clicks.

  1. Select: Right-click a project, folder, or multiple files in the Package Explorer.
  2. Execute: Select ðŸ¤– Copy Source for AI from the context menu.
  3. Adjust: A configuration dialog appears. You can adjust filters and size limits for the current export session.
  4. Save: Click OK and specify a location to generate the merged source_context.txt file.

File & Folder Filtering

Optimize your prompt and save tokens by excluding unnecessary data (binaries, build artifacts, etc.).

Allowed Extensions (Whitelist)

Specify extensions like java, py, js to collect only matching files. If empty, all files are targeted.

Excluded Extensions (Blacklist)

Define extensions like class, jar, exe to exclude them from the analysis.

Folder Exclusion: You can skip directories such as target, bin, .git by specifying their names.

Output Size Limits (MB)

Prevent exceeding AI service input limits (context window) by restricting the maximum size of the output file.

  • MB Units: Entering 2 in the dialog will automatically stop writing once the file reaches approximately 2MB.
  • Safe Truncation: If the limit is reached, a warning message is inserted at the end of the file indicating which files were processed.
  • Unlimited Output: Enter 0 to export all selected project content without any size restrictions.

Encoding & Format

Automatic UTF-8 Unification:
Regardless of the original file encoding (Shift-JIS, EUC-JP, etc.), the exporter reads files based on Eclipse's charset settings and converts everything into a UTF-8 encoded file. This prevents garbled text in AI analysis.

The exported text follows a structure where each file path is marked with a Markdown header (##) and the content is wrapped in code blocks (```), making it easy for the AI to understand the file structure.
